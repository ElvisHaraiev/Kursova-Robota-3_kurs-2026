# Курсова Робота: Автоматизована інформаційна система «MY RESTAURANT»
«MY RESTAURANT» — це сучасний десктопний застосунок для комплексного управління ресторанним бізнесом та службою доставки. Система розроблена для автоматизації роботи персоналу (офіціантів, касирів), кухарів та адміністраторів. Вона дозволяє пришвидшити обслуговування клієнтів, усунути помилки при передачі замовлень на кухню, вести клієнтську базу (CRM) та формувати точну фінансову аналітику.

# 🛠 Технологічний стек
Мова програмування: C#

Платформа: .NET Framework (Windows Forms)

База даних: MySQL (взаємодія через низькорівневий ADO.NET)

UI/UX Дизайн: Кастомний інтерфейс (Flat Design) з програмною відмальовкою елементів (заокруглення, градієнти) за допомогою System.Drawing.Drawing2D (GDI+).

Архітектура та Патерни: * RBAC (Role-Based Access Control): Багаторівнева система контролю доступу.

Strategy Pattern: Гнучка архітектура для динамічного розрахунку знижок (промокоди, нові клієнти).

Auto-Initialization: Автоматичне розгортання структури бази даних при першому запуску.

Асинхронність: Використання Task.Run для фонових процесів (таймери готовності на кухні).

# 🚀 Основні можливості (Фічі)
Автоматичне створення БД (Code-First підхід): Програма не потребує ручного імпорту таблиць. При першому запуску система сама створює базу __restaurant_db,__ всі 7 необхідних таблиць та генерує обліковий запис адміністратора.

Рольова модель доступу (RBAC): Жорстке розділення прав між ролями: Admin (повний доступ), Cook (тільки кухня та стоп-лист), Personal (каса, зали, доставка).

Модуль Доставки та CRM: Автоматична перевірка клієнта за номером телефону. Якщо клієнт новий — система реєструє його в базі та автоматично застосовує знижку 20%.

Гнучка система знижок (Strategy): Розділена бізнес-логіка для застосування промокодів (наприклад, WELCOME2026) та акцій без зміни основного коду каси.

Кухонний модуль (Live Queue): Екран для кухарів із живою чергою замовлень. Зміна статусів (Pending ➔ Cooking ➔ Ready) супроводжується системними сповіщеннями для офіціантів.

Аналітика (Дашборд): Зведені фінансові показники, статистика замовлень та агреговані SQL-запити для власників бізнесу.

# ⚙️ Встановлення та налаштування (Швидкий старт)
Завдяки вбудованому механізму автоматичної ініціалізації (DbHelper.InitializeDatabase()), запуск проєкту максимально спрощено!

Крок 1. Налаштування сервера MySQL

Завантажте та встановіть локальний сервер (рекомендується XAMPP).

Відкрийте панель керування XAMPP та натисніть Start біля модуля MySQL.

Більше нічого робити не потрібно! Програма сама створить базу даних.

Крок 2. Відновлення демонстраційних даних (За бажанням)
Якщо ви хочете, щоб програма запустилася не порожньою, а з готовим меню (стравами), клієнтами та історією замовлень:

Відкрийте брайзер і перейдіть за адресою: http://localhost/phpmyadmin/

Створіть порожню базу з назвою __restaurant_db__ (Кодування: utf8mb4_unicode_ci).

Перейдіть у вкладку Import (Імпорт) і завантажте файл __restaurant_db.sql__ з папки проєкту.

# 💻 Як запустити проєкт
Спосіб 1: Через Visual Studio (Для розробки)

Відкрийте файл рішення (.sln) у Visual Studio.

Переконайтеся, що XAMPP (MySQL) увімкнений.

Натисніть кнопку Start або клавішу F5.

Спосіб 2: Як готову програму (.exe)

Переконайтеся, що XAMPP (MySQL) увімкнений.

Перейдіть у директорію проєкту: \bin\Debug\ (або \Release\).

Запустіть файл Курсова.exe.S

# 📖 Сценарії використання (Інструкція користувача)
🔐 1. Авторизація
При першому (чистому) запуску база даних створюється автоматично.
Для входу використовуйте згенерований акаунт адміністратора:

# Логін: admin

# Пароль: 1234
(Увійшовши як адміністратор, ви зможете створити акаунти для кухарів та офіціантів).

🤵 2. Сценарій: Офіціант / Касир (Роль: Personal)
Оформлення замовлення: Перейдіть у модуль столів або доставки. Додайте страви до кошика.

Робота з CRM: При оформленні доставки введіть номер телефону. Система перевірить, чи є клієнт у базі, і якщо він новий — автоматично застосує стратегію ## NewCustomerDiscount (знижка 20%).

Промокоди: Застосуйте код (напр. WELCOME2026). Завдяки патерну Strategy сума перерахується миттєво.

Після підтвердження замовлення відправляється в БД зі статусом Pending (Очікує).

👨‍🍳 3. Сценарій: Кухар (Роль: Cook)
Після авторизації кухар потрапляє виключно у свій ізольований модуль (інші меню приховані).

На екрані з'являються картки нових замовлень від офіціантів.

Кухар змінює статус: бере в роботу (Cooking), а після приготування натискає кнопку Готово (Ready).

Асинхронний зв'язок: Офіціант автоматично отримає спливаюче повідомлення про те, що страву можна забирати.

Стоп-лист: Кухар може вимикати відображення страв, інгредієнти для яких закінчилися.

👔 4. Сценарій: Адміністратор (Роль: Admin)
Повний контроль: Доступ до всіх функцій системи.

Аналітична панель (Dashboard): Відстеження прибутку, кількості чеків та популярних страв за обраний період (агрегація даних через MySQL).

Управління: Додавання нових позицій у меню, зміна цін, реєстрація нових працівників (Офіціантів/Кухарів) та управління базою клієнтів.
