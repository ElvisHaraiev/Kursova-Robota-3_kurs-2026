# 🍽 Курсова Робота: Автоматизована інформаційна система «MY RESTAURANT»
«MY RESTAURANT» — це професійний десктопний застосунок для повного циклу управління ресторанним бізнесом. Система автоматизує взаємодію між залом, кухнею та адміністрацією, мінімізуючи людський фактор та забезпечуючи високу швидкість обслуговування. Проєкт поєднує в собі CRM-систему, касовий апарат, кухонний дисплей та аналітичний інструмент.

# 🛠 Технологічний стек
Мова програмування: C#

Платформа: .NET Framework (Windows Forms)

База даних: MySQL (ADO.NET)

Графічний інтерфейс: Custom UI на базі System.Drawing.Drawing2D (градієнти, закруглені кути, динамічна верстка).

Архітектурні патерни: * RBAC (Role-Based Access Control): Розмежування прав на рівні об'єктів.

Strategy Pattern: Розрахунок дисконтної сітки.

Singleton/Static Helpers: Для глобального доступу до БД та безпеки.

# 🔐 Безпека та технічні стандарти (Senior Level)
Проєкт реалізований із дотриманням стандартів промислової розробки:

Криптографія (SHA-256): Паролі користувачів ніколи не зберігаються у відкритому тексті. При реєстрації та вході використовується незворотне хешування.

Фінансова точність: Для всіх грошових операцій (ціни, суми, знижки) замість типу double використано decimal (10,2), що виключає помилки округлення при розрахунках.

Реляційна цілісність: База даних використовує Foreign Keys та обмеження ON DELETE CASCADE, що забезпечує автоматичне очищення пов'язаних даних.

Безпека запитів: Використання параметризованих запитів (MySqlParameter) для повного захисту від SQL-ін'єкцій.

# 🚀 Основні можливості (Фічі)
Code-First Ініціалізація: Повний автоматичний запуск. Програма сама створює базу restaurant_db та 7 таблиць при першому старті.

Smart CRM: Система впізнає клієнтів за номером телефону, веде історію їхніх замовлень та автоматично нараховує знижку "Нового клієнта" (20%).

Інтерактивна Кухня: Жива черга замовлень для кухаря зі статусами приготування та автоматичними асинхронними повідомленнями для персоналу в залі.

Управління Столами: Візуальний контроль зайнятості столів та можливість дозамовлення на відкритий чек.

# 📊 Структура бази даних
Система базується на 7 взаємопов'язаних таблицях:

users — персонал та захищені паролі.

categories — дерево категорій меню.

menuitems — страви, ціни та наявність.

orders & order_items — головний модуль продажів.

clients — база CRM (клієнти).

reservations — календар бронювань столиків.

# ⚙️ Налаштування бази даних та запуск
Сервер: Запустіть MySQL через XAMPP або інший локальний сервер.

Підключення: Переконайтеся, що порт 3306 відкритий.

Запуск: Просто запустіть Курсова.exe (або скомпілюйте проєкт). Метод DbHelper.InitializeDatabase() автоматично розгорне всю інфраструктуру.

Хешування: Якщо ви вносите дані вручну через phpMyAdmin, пам'ятайте, що у стовпець password_hash потрібно вставляти 64-символьний хеш (SHA-256), а не звичайний текст.

📖 Інструкція та ролі користувачів
🔐 Вхід у систему за замовчуванням:

Admin: Логін admin, Пароль 1234 (надає доступ до аналітики та керування акаунтами).

Kitchen: Логін Kitchen, Пароль 1212 (тільки кухня та меню).

🤵 Роль: Персонал (Waiter / Personal)
Доступ до модулів: Столи, Доставка, Історія, Бронювання. Основна функція — швидке формування чека та робота з клієнтами.

👨‍🍳 Роль: Кухар (Cook / Kitchen)
Обмежений доступ: Кухар бачить виключно модулі «Кухня» та «Меню». Це гарантує, що виробничий персонал не має доступу до фінансової звітності чи особистих даних інших працівників (може змінювати "Стоп-лист").

👔 Роль: Адміністратор (Admin)
Доступ до всіх гілок системи, включаючи «Аналітику» (звіти про прибуток) та «Керування акаунтами», де реалізовано механізм безпечного створення нових користувачів.

# 🛠 Вирішення типових проблем (Troubleshooting)
При розгортанні проєкту на новому комп'ютері можуть виникнути локальні технічні нюанси. Нижче наведено список частих запитань та рішень:

🔴 1. Проблема: Помилки компіляції (The type or namespace name 'MySql' could not be found)
Причина: У проєкті використовується зовнішня бібліотека MySql.Data для роботи з базою даних, і Visual Studio не завантажила її автоматично.

Рішення: 1. Відкрийте проєкт у Visual Studio.
2. У вікні Solution Explorer натисніть правою кнопкою миші на рішення (Solution).
3. Виберіть Restore NuGet Packages (Відновити пакети NuGet).
4. Перезапустіть збірку проєкту (Ctrl+Shift+B).

🔴 2. Проблема: Програма крашиться при запуску або видає помилку підключення до БД
Причина: Програма не може підключитися до вашого локального MySQL сервера. За замовчуванням у класі DbHelper.cs прописано стандартний рядок підключення без пароля: Uid=root;Pwd=;.

Рішення:

Переконайтеся, що у вас запущено сервер (наприклад, світиться зелений індикатор MySQL у панелі XAMPP).

Якщо ваш локальний MySQL сервер має пароль для користувача root, перейдіть у файл DbHelper.cs та змініть статичну змінну connectionString, вписавши свій пароль: Pwd=ВашПароль;.

🔴 3. Проблема: Інтерфейс виглядає "з'їхавшим", елементи налізають один на одного
Причина: Технологія Windows Forms чутлива до системного масштабування екрана (DPI Scaling) у Windows, особливо на сучасних ноутбуках (де масштаб часто стоїть 125% або 150%).

Рішення: Для ідеального відображення Custom UI (заокруглених кнопок та панелей) рекомендується:

Натиснути правою кнопкою миші на робочий стіл Windows -> Параметри дисплея.

Змінити Масштаб та розмір тексту на 100%.

Перезапустити програму.

🔴 4. Проблема: Замість картинок страв відображаються порожні квадрати
Причина: Програма шукає фізичні файли зображень, шляхи до яких зберігаються в базі даних (image_path).

Рішення: Переконайтеся, що папка із зображеннями лежить у правильній директорії (зазвичай це bin/Debug/Images або відповідна папка ресурсів).
