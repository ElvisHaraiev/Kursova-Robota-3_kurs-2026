# Курсова Робота
Автоматизована інформаційна система «MY RESTAURANT»
«MY RESTAURANT» — це десктопний застосунок для комплексного управління ресторанним бізнесом. Система призначена для автоматизації роботи офіціантів, кухарів та адміністраторів, що дозволяє пришвидшити обслуговування клієнтів, мінімізувати помилки в замовленнях та вести точний фінансовий облік.

🛠 Технологічний стек
Мова програмування: C#

Платформа: .NET Framework (Windows Forms)

База даних: MySQL (взаємодія через ADO.NET)

Архітектура: Модульна структура з використанням патерну проєктування Strategy (для розрахунку знижок) та рольової моделі RBAC.

🚀 Основні можливості (Фічі)
Рольова модель доступу (RBAC): Розділення прав між Адміністратором, Офіціантом та Кухарем.

Управління замовленнями (Кошик): Зручний інтерфейс для офіціантів для додавання страв, розрахунку суми та оформлення.

Гнучка система знижок: Автоматичний розрахунок фінальної суми (знижки для нових клієнтів, промокоди на кшталт WELCOME2026).

Кухонний модуль: Екран для кухарів із чергою замовлень у реальному часі.

Аналітика (Дашборд): Зведені фінансові показники та статистика замовлень для власників бізнесу.

⚙️ Встановлення та налаштування бази даних
Щоб запустити проєкт на вашому локальному комп'ютері, виконайте наступні кроки:

Встановіть СУБД: Переконайтеся, що у вас встановлено локальний сервер MySQL (наприклад, XAMPP, WAMP або окремий MySQL Server).

Створіть базу даних: * Відкрийте phpMyAdmin або MySQL Workbench.

Виконайте SQL-скрипт database_setup.sql (який знаходиться в папці проєкту) для створення бази restaurant_db та всіх необхідних таблиць.

Налаштуйте підключення (Connection String):

Відкрийте проєкт у Visual Studio.

Перейдіть до файлу DbHelper.cs.

Переконайтеся, що connectionString відповідає вашим налаштуванням БД (користувач root, без пароля за замовчуванням):

C#

Запустіть проєкт: Натисніть F5 або кнопку "Start" у Visual Studio.

📖 Як користуватися застосунком
Програма має різні сценарії використання залежно від того, під яким акаунтом ви увійшли.

1. Авторизація (Вхід у систему)
При запуску програми з'являється вікно логіну. Введіть логін та пароль. Система автоматично визначить вашу роль і приховає або покаже відповідні кнопки.
(Для тестування можна створити в БД користувачів з ролями admin, waiter, cook).

2. Сценарій: Офіціант (Waiter)
Створення замовлення: Перейдіть у розділ меню. Оберіть номер столика або вкажіть "Доставка".

Додавання в кошик: Клікайте на страви з меню — вони автоматично з'являтимуться в кошику (DataGrid). Якщо клікнути двічі, кількість (Quantity) збільшиться.

Оплата та знижки: Перед оплатою оберіть тип клієнта або введіть промокод. Система (завдяки патерну Strategy) автоматично перерахує загальну суму.

Натисніть «Підтвердити замовлення». Воно відправиться в базу даних і з'явиться на екрані кухаря.

3. Сценарій: Кухар (Cook)
Увійшовши як кухар, ви побачите лише модуль "Кухня / Черга замовлень".

На екрані відображаються нові замовлення зі статусом pending (очікує).

Коли починаєте готувати, змініть статус на cooking (готується).

По завершенню натисніть кнопку «Готово» (статус зміниться на ready), і офіціант побачить, що страву можна забирати.

4. Сценарій: Адміністратор (Admin)
Адміністратор має повний доступ до всіх модулів.

Дашборд (Аналітика): Перегляд загального доходу, кількості чеків за день та розподілу оплат (готівка/картка).

Управління: Можливість додавати нові страви в меню, змінювати ціни, реєструвати нових працівників або блокувати звільнених.
